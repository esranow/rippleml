# v0.2.0 Speed House — Technical Documentation

## Architecture Overview

v0.2.0 replaces the manual PyTorch training loop with a PyTorch Lightning 
backbone while maintaining full backward compatibility via graceful fallback.

## rp.compile()

Wraps torch.compile with platform detection. Falls back to eager mode on 
Windows and CUDA < 11.7 with a warning rather than crashing.

## rp.run()

Master execution function. Internally:
1. Instantiates AutoScaler from domain/equation bounds
2. Wraps model + physics in LightningEngine
3. Configures pl.Trainer with strategy/devices/precision
4. Generates Sobol collocation DataLoader from domain
5. Runs trainer.fit()
6. Returns model state dict + scaler state

## Adam → L-BFGS Handoff

The LightningEngine monitors loss history with a sliding window of 200 steps.
When rel_std = std(window) / mean(window) < 1e-3, training switches to L-BFGS.
This preserves Adam's momentum basin while allowing L-BFGS to converge precisely.

Proven results:
- Wave 1D: L2=8.38e-05 (vs 8.77e-01 without handoff)
- Heat 1D: L2=3.80e-04

## AutoScaler Chain Rule

For u_hat = u/U0, x_hat = x/L0:
- du/dx = (U0/L0) * du_hat/dx_hat
- d²u/dx² = (U0/L0²) * d²u_hat/dx_hat²
- du/dt = (U0/T0) * du_hat/dt_hat

Applied invisibly in training_step before PDE residual evaluation.

## Auto-Migrate

Static AST analysis. Parses DeepXDE scripts and extracts:
- PDE function signature and body
- Domain type and bounds
- Boundary and initial conditions
- Collocation point counts

Outputs rippl equivalent with TODO markers where manual operator 
composition is required. Never executes foreign code.

## DDP Multi-GPU

Requires: pip install rippl[distributed]
Usage: rp.run(..., strategy="ddp", devices=4)
Lightning handles process spawning, gradient synchronization, and 
checkpoint coordination automatically.
